COTIZADOR PM/CP - FUNCIONAMIENTO GENERAL DEL CODIGO
Ultima actualizacion: 2026-03-28

Este documento resume la arquitectura actual del proyecto despues de la reorganizacion
del repositorio.

===============================================================================
1. ESTRUCTURA DEL REPOSITORIO
===============================================================================

- `frontend/`
  contiene HTML, JS, CSS, assets y carpeta publica minima
- `backend/`
  contiene PocketBase, hooks, migraciones, data y logs
- `development/`
  scripts para levantar backend y frontend localmente
- `production/`
  servicio Windows, runner y utilidades de despliegue
- `docs/`
  documentacion tecnica vigente

===============================================================================
2. FRONTEND
===============================================================================

Raiz funcional:

- `frontend/client/`

Entradas principales:

- `frontend/client/index.html`
- `frontend/client/cotizador/`
- `frontend/client/cotizadorcp/`
- `frontend/client/public/`
- `frontend/client/system/users1.html`

Core compartido:

- `frontend/client/js/hub-config.js`
- `frontend/client/js/layout.js`
- `frontend/client/services/*.js`

Assets compartidos:

- `frontend/assets/`

Publicacion estatica para produccion:

- `frontend/pb_public/`

===============================================================================
3. BACKEND
===============================================================================

- `backend/pocketbase.exe`
- `backend/pb_hooks/`
- `backend/pb_migrations/`
- `backend/pb_data/`
- `backend/logs/`

PocketBase es el backend principal y centraliza auth, datos, storage y hooks.

===============================================================================
4. OPERACION
===============================================================================

Desarrollo:

- `development/dev-start.bat`
- `development/frontend-dev-start.bat`

Produccion / Windows:

- `production/backend-service.bat`
- `production/levantar-todo.bat`
- `production/deploy/run-pocketbase-service.ps1`

10. DONDE TOCAR SEGUN EL CAMBIO
===============================================================================

- backend URL o branding:
  `frontend/client/config/hub-runtime.json`
  `frontend/client/js/hub-config.js`

- layout, sesion o permisos:
  `frontend/client/js/layout.js`

- cliente base PocketBase:
  `frontend/client/services/pb-core.js`

- reglas publicas de cotizaciones:
  `backend/pb_hooks/10_cotizaciones.pb.js`

- despliegue Windows:
  `production/backend-service.bat`
  `production/deploy/backend-service.local.conf`

===============================================================================
11. PUNTOS DE CUIDADO
===============================================================================

- `backend/pb_data/` es estado vivo
- no mover `frontend/assets/` sin revisar rutas relativas del HTML
- no cambiar PDFs sin validar preview, descarga y documento guardado
- cualquier cambio multi-tenant debe respetar `tenant`
